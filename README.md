## 📁 Example Profile Configuration

Below is a sample profile for **archival film restoration** — optimized for 16mm scans with 2.8K resolution and heavy grain.

```ini
[profile:archival_16mm]
name = "Archival Film 16mm (2026)"
resolution = 2840x2160
temporal_radius = 3
spatial_threshold = 0.42
temporal_threshold = 0.18
chroma_smoothing = medium
noise_profile = automatic
gpu_acceleration = force_cuda
output_format = prores_4444_alpha
multilingual_metadata = true
```

### Explanation of Key Parameters

- **`temporal_radius = 3`**: Balances frame-to-frame consistency without introducing ghosting — ideal for shaky archival material.
- **`spatial_threshold = 0.42`**: Aggressive spatial denoising that respects edge detail (tested with Kodak Vision3 stock).
- **`multilingual_metadata = true`**: Embeds language tags in the output file for downstream localization pipelines.

## 🤖 AI Integration: OpenAI & Claude API

Noise is not random — it follows patterns. **But which patterns?** This is where AI integration shines.

### OpenAI API (GPT-4o)
- **Use case**: Real-time noise type classification  
- **Example**: The system sends a downsampled frame to OpenAI, which returns a JSON payload:  
  ```json
  { "noise_type": "gaussian_salt_pepper", "confidence": 0.94, "recommended_profile": "digital_high_gain" }
  ```
- Automatically updates the active profile without manual intervention.

### Claude API (Claude 3.5 Sonnet)
- **Use case**: Semantic understanding of footage context  
- **Example**: A sentence-level analysis of the scene — "low-light indoor, high ISO, fluorescent flicker" — triggers a specialized temporal flicker reduction pass before denoising.

**To enable**: Set environment variables `OPENAI_API_KEY` and `CLAUDE_API_KEY` in your shell profile, or pass them directly in the CLI invocation.

## 📱 Responsive UI & 24/7 Support Architecture

### Responsive UI Overlays

The plugin interface adapts to different screen sizes and input methods:

- **Desktop (1920x1080+)**: Full parameter sliders, histograms, and split-screen previews.
- **Tablet (1024x768)**: Collapsible panels with gesture-based slider control (swipe up/down for values).
- **Phone (480x320)**: Minimalist view with essential controls — threshold, radius, and preview toggle.

The UI is rendered using a custom WebAssembly-based overlay (no Electron), ensuring <5ms UI latency even on 4K timelines.

### 24/7 Customer Support Integration

We embed a **lightweight support portal** directly into the plugin:

```python
# Internal support bridge (simplified)
class SupportBridge:
    def __init__(self):
        self.knowledge_base = load_embeddings("support_docs_2026.bin")
        self.ticket_queue = Queue()
    
    def auto_resolve(self, error_code):
        if error_code in self.knowledge_base:
            return self.knowledge_base[error_code]["solution"]
        else:
            self.ticket_queue.enqueue(error_code)
            return "Escalated to human team (SLA: 2 hours)"
```

The bridge automatically categorizes tickets by severity:
- **P0 (Critical)**: Plugin crash → immediate notification to core team via Slack webhook.
- **P1 (High)**: Profile corruption → auto-rollback to last stable backup.
- **P2 (Medium)**: CLI parameter mismatch → suggests correct syntax.
